# Railway Deploy: Test + Production

Этот проект настроен для запуска на Railway без Docker через Nixpacks (`railway.json` + `nixpacks.toml`).

## Целевая схема

- Проект: `smart_b12`
- Окружения:
  - `production`
  - `test`
- Сервисы (рекомендуется):
  - `smart_b12-prod` (ветка деплоя: `main` или `prod`)
  - `smart_b12-test` (ветка деплоя: `test`)
- Базы данных:
  - отдельная БД для `production`
  - отдельная БД для `test`

---

## 1. Создайте проект и окружения в Railway

1. Создайте проект `smart_b12`.
2. Создайте/оставьте два окружения в Railway:
   - `production`
   - `test`

---

## 2. Создайте 2 сервиса из одного репозитория

Создайте два сервиса в рамках одного Railway-проекта:

- Сервис `smart_b12-prod`
  - Source repo: этот репозиторий
  - Ветка: `main` (или `prod`, если используете отдельную продовую ветку)
- Сервис `smart_b12-test`
  - Source repo: этот репозиторий
  - Ветка: `test`

Оба сервиса используют корневые `railway.json` и `nixpacks.toml`.

---

## 3. Настройте переменные для каждого окружения

Задайте переменные отдельно для каждой пары сервис/окружение.

## Обязательные

- `DATABASE_URL` (должен отличаться для prod/test)
- `SECRET_KEY` (должен отличаться для prod/test)

## Рекомендуемые

- `ENV=production` для prod-сервиса
- `ENV=test` для test-сервиса
- `CORS_ORIGINS=https://<your-prod-domain>` для prod
- `CORS_ORIGINS=https://<your-test-domain>` для test

`PORT` Railway задает автоматически.

---

## 4. Домены

Назначьте отдельные публичные домены:

- prod домен -> `smart_b12-prod`
- test домен -> `smart_b12-test`

---

## 5. Поток деплоя

- Push в `main`/`prod` -> деплой `smart_b12-prod`
- Push в `test` -> деплой `smart_b12-test`

---

## 6. Чеклист проверки

Для каждого сервиса:

1. Откройте `/health` -> должен вернуться `{"status":"ok"}`.
2. Откройте `/` -> фронтенд должен быть доступен.
3. Выполните логин и проверьте API-запросы.
4. Убедитесь, что используется правильная БД (prod/test).

---

## Примечания

- Стартовый скрипт `./railway-start.sh` запускает Alembic миграции при старте сервиса.
- Разделяйте креды продовой и тестовой БД.
